# 使数组唯一的最小增量

# 题目内容

给定整数数组 A，每次 move 操作将会选择任意 A[i]，并将其递增 1。

返回使 A 中的每个值都是唯一的最少操作次数。

示例 1:

输入：[1,2,2]

输出：1

解释：经过一次 move 操作，数组将变为 [1, 2, 3]。

示例 2:

输入：[3,2,1,2,1,7]

输出：6

解释：经过 6 次 move 操作，数组将变为 [3, 4, 1, 2, 5, 7]。可以看出 5 次或 5 次以下的 move 操作是不能让数组的每个值唯一的。 提示： 0 <= A.length <= 40000

0 <= A[i] < 40000

# 个人理解

  • 每次只能使其中一个数加 1
  • 每个数最少差 1
/**
 * 1.先进性排序
 * 2. 检测后一个值和前一个值插值大于1，就修改前一个值
 *
 * @param A
 * @return
 */
public static int minIncrementForUnique(int[] A)
    // 排序
    Arrays.sort(A);
    System.out.println(Arrays.toString(A));
    // 记录总共执行的次数
    int count = 0;
    // 遍历
    for (int i = 1; i < A.length; i++) {
        // 如果后一个数小于等于前一个数，需要后一个数添加为比钱一个数大1
        if (A[i] <= A[i - 1]) {
            // 判断后一个数比钱一个数大1需要执行的次数（也是需要添加的数据）
            int temp = A[i - 1] - A[i] + 1;
            A[i] += temp;
            count += temp;
        }
    }
    System.out.println(Arrays.toString(A));
    return count;
}
  • 虽然该方法能够实现，但是消耗的时间却太多，落后于 80%。

# 官方的解法理解

/**
 * 该算法的简介：
 * <p> 如果一个从 X-->Y，另一个数从 M-->N,和 X-->N,M-->Y需要执行的次数是一致的。变成公式即：(Y+N)-(X+M)
 * 1.构建最大的数组
 * 2.遍历构建的数组，通过设置第几位的数值，标志第几个树重复的次数
 * 3.并且记录当前重复的数据（重复的个数 * 当前数值）
 * 4.如果不重复，当前的值为0，说明不存在该数，然后判断是否有重复，如果有，则需要去除一个重复的，加上当前数值，进而实现 (Y - X)
 *
 * @param A
 * @return
 */
public static int minIncrementForUnique1(int[] A) {
    int[] count = new int[80000];
    for (int x : A) count[x]++;

    int ans = 0, taken = 0;

    for (int x = 0; x < 80000; ++x) {
        // 判断是否重复
        if (count[x] >= 2) {
            // 如果重复 把token记录现在已经有的重复的个数
            taken += count[x] - 1;
            // 直接记录重复的数据
            ans -= x * (count[x] - 1);
        } else if (taken > 0 && count[x] == 0) {
            taken--;
            ans += x;
        }
    }

    return ans;
}
