# RPC
RPC是远程调用，即本地调用与远程。其本质通过socket进行调用。

> 1. 服务注册到注册中心（注册服务的ip和host），同时缓存（bean, method）
> 2. 注册中心返回相关信息
> 3. 客户端通过注册中心返回的数据进行socket请求，并获取数据

![RPC调用图](http://notebook.zhangjiahao.site/markdown-img-paste-20200412140644527.png)

RPC总共包括一下模块：

> - 序列化和反序列化
> - 协议模块
> - 数据传输
> - 工具（反射创建对象、获得公有方法等等）
> - server
> - client

## 序列化和反序列化

通过fastJson可实现对象二进制之间的相互转化，防止数据传输过程导致数据格式问题

## 协议

通过自定义协议，实现数据的请求和返回的规范

## 数据传输

进行数据的转化

## 工具

注册的为类的公有方法，所以需通过类获得所有的公有方法，也需要通过类获取对象，从而在远程调用时创建对应的对象执行相应的操作。

## server

服务端，编写测试提供服务方

## client

客户端，编写调用远程方法
