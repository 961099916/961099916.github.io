## SSO单点登陆的理解

* [SSO单点的登录](https://baike.baidu.com/item/SSO/3451380?fr=aladdin)可以让开发出来的不同系统只需登陆一次，方便了用户的使用，犹如一卡通，只需一次登陆则可在各个系统中使用，对于开发者也减少了开发量，无需每个系统都需要重新开发用户验证登陆模块，对于安全问题也大大的增加了其风险，如果其中一个系统安全性能较低，如果在此破解，则会导致各个系统都被破解。

## 2. SSO同域名登陆

* 同域名的单点登录其实就是在根域名上加上Cookie，使得不同的系统都能够进行获得Cookie，关键代码：

``` java
 // 设置Cookie的路径为根域名上
 cookie.setPath("/");
```

## 3. SSO同父域登陆

* 同父域的单点登录比同域名单点登录麻烦点，需要在父级域名上添加Cookie，因为涉及到不同域名，所以之间的通信需要HTTP通信进行判断。

关键代码：

``` java
// 设置在父级域名上
cookie.setDomain("x.com");
// 设置在根路径
cookie.setPath("/");
```

## 4. SSO跨域登陆

* 这个相比较更加困难，但是起原理还是听简单的，由于不同的域名所以不能单单的在某个域名上添加Cookie，所以解决办法就是在每个域名上都添加Cookie。可以通过创建隐藏的\<iframe\>标签来调用不同域名的添加Cookie的服务来实现跨域的单点登录
