# 数据仓库
## 介绍
- 开发过程中，不免会进行许多的数据收集，采集大量的数据，在此之前仅仅只是把采集的数据进行放置数据库中，但在最近工作中用到了数据仓库。
- 数据仓库，对于一个企业也是相当重要的，可通过自家产品的相关数据进行规划进一步的发展，收集网络中的数据，寻求产品的不足和长处，从而能够让产品更加完善。
## 数据仓库架构
数据仓库的整体架构为一下图：
![数据仓库](http://notebook.zhangjiahao.site/markdown-img-paste-20200314141152533.png)
1. 数据的收集。只需把所有的数据进行保存到数据库，无需进行太多的操作，保存源数据。
2. 数据清洗。源数据（ODS）层的数据由于是原始数据，包含很多的重复，错误的数据，所以需要经过清洗，从而去除重复和无效的数据。
3. 数据分发。经过清洗之后数据将提高其准确性，但是应用会采用不同的数据，所以需要把所需要的数据从数据仓库中提取到自己的数据库中进行进一步的展示。
## 项目中的使用
- 由于处于疫情阶段，公司开展了疫情看板，所以采集了大量的疫情相关的数据，进而达到对疫情进行全面的认识。
- 我们采集了新浪、腾讯的疫情相关的数据，以供实时展示，由于之前未清洗数据导致数据不准确和SQL复杂度大大增加。于是采用了该架构。从而大大的降低了SQL的复杂度和提高了数据的准确。我们也进行人工采集卫检委数据，进行人工兜底，从而使得T-1数据的更加准确。
